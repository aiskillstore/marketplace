# jstodocx.com API Reference

Base endpoint:

```text
POST https://jstodocx.com/api/v1/docx
```

Response content type:

```text
application/vnd.openxmlformats-officedocument.wordprocessingml.document
```

The response is a binary `.docx` file. It includes `content-disposition`, `cache-control: no-store`, and rate-limit headers.

## Authentication

The endpoint may be public or may require a key, depending on site configuration.

Accepted API key headers:

```text
Authorization: Bearer YOUR_API_KEY
x-api-key: YOUR_API_KEY
```

Never print, save, or commit API keys. If a request returns `401`, ask the user for a key or stop.

## Rate Limits

Typical response headers:

```text
x-ratelimit-limit: 30
x-ratelimit-remaining: 29
x-ratelimit-reset: 3600
```

If the response is `429`, wait or ask the user whether to retry later. Do not loop aggressively.

## Mode: html

Use this when the user has basic document HTML.

Request:

```json
{
  "mode": "html",
  "title": "Release Plan",
  "html": "<h1>Release Plan</h1><p>Hello <strong>team</strong>.</p><ul><li>Ship export</li></ul>"
}
```

Supported formatting is intentionally basic: headings, paragraphs, bold, italic, underline, inline code, lists, preformatted text, and simple tables. Unsafe blocks such as scripts, styles, iframes, objects, embeds, and templates are ignored.

## Mode: js-source

Use this when the user wants to put JavaScript source code into a Word document as text.

Request:

```json
{
  "mode": "js-source",
  "title": "Widget Export Code",
  "code": "function exportWidget() {\n  return 42;\n}",
  "options": {
    "lineNumbers": true
  }
}
```

This mode does not run JavaScript. It formats source text into a DOCX archive with optional line numbers.

## Mode: template

Use this for controlled document generation from structured fields.

Request:

```json
{
  "mode": "template",
  "template": "invoice",
  "title": "Invoice API Draft",
  "fields": {
    "heading": "Invoice for Northwind",
    "paragraph": "Generated from controlled API fields.",
    "bullets": ["Net 14 payment terms", "Prepared for review"],
    "rows": [["Item", "Qty", "Amount"], ["Implementation", "12", "$1,080"]],
    "footer": "Generated by jstodocx.com API"
  }
}
```

Supported template names:

- `report`
- `invoice`
- `proposal`
- `resume`
- `meeting-notes`

## Errors

Common status codes:

- `400`: invalid JSON, unsupported mode, or missing content.
- `401`: API key is required or invalid.
- `413`: request body is too large.
- `415`: use `application/json`.
- `429`: rate limit exceeded.
- `500`: DOCX generation failed.

## Validation

After saving the response:

```bash
file output.docx
unzip -p output.docx word/document.xml | head
```

A valid DOCX is a ZIP archive and usually begins with bytes `PK`.
