# 政务服务模糊问题判断规则

本文档定义了政务服务场景中模糊问题的判断标准和输出规范。

---

## 核心原则

**判断目标**：识别用户问题中的**业务客体（具体办什么事/查什么政策）**是否达到了可以进入系统检索的精确度。

**输出结果**：
- `1` - 非模糊问题：业务名称具体、事项指向明确
- `0` - 模糊问题：指向不明或信息缺失

---

## 判断标准

### 非模糊问题（输出 1）

**核心特征**：具备"唯一性"或"专业性"

#### 判断条件

**条件 1：包含标准业务全称或专业术语**
- 示例：
  - "科技型中小企业评价"
  - "创新积分排名"
  - "医师备案"
  - "执业类别变更"
  - "高新技术企业认定"

**条件 2：具备"具体限定词 + 泛称"结构**
- 示例：
  - "食品经营许可证"（限定词：食品经营）
  - "不动产权证"（限定词：不动产）
  - "灵活就业人员社保补贴"（限定词：灵活就业人员社保）

**判断逻辑**：即便跨委办局，只要能通过问题直接查到对应的办事流程或政策文件，即为非模糊。

#### 特殊案例（直接判断为非模糊）
- 用电业务办理
- 婚内赠与
- 企业备案

---

### 模糊问题（输出 0）

**核心特征**：存在"指向不明"或"信息缺失"

#### 判断条件

**条件 1：仅提供办事场景，未指明具体证件或业务**

示例：
- "开店的许可证" → 开店可能涉及食品、烟草、消防等多种许可
- "买房的证" → 可能是不动产权证、契税完税证明等
- "孩子上学的证明" → 可能是居住证明、社保缴纳证明等

**条件 2：使用不带修饰的泛称**

示例：
- "许可证延续" → 哪类许可证？
- "许可证书补领" → 哪类证书？
- "政策补贴领取" → 哪项政策？
- "证书怎么办" → 什么证书？

**条件 3：主语指向过大**

示例：
- "政府的扶持政策" → 哪个部门？哪类政策？
- "科技局的项目" → 具体是什么项目？

---

## 判断流程图

```
用户问题输入
    ↓
是否包含业务客体？
    ↓ 否 → 输出 0（完全无关或无业务）
    ↓ 是
业务客体是否具体？
    ↓
检查判断条件：
    ├─ 包含标准业务全称或专业术语？
    │   ↓ 是 → 输出 1
    │   ↓ 否
    ├─ 具备"具体限定词 + 泛称"结构？
    │   ↓ 是 → 输出 1
    │   ↓ 否
    └─ 是否为场景描述或泛称？
        ↓ 是 → 输出 0
        ↓ 否 → 输出 1（特殊情况）
```

## 边界情况处理

### 需要进一步判断的情况

**情况 1：限定词不够具体**
- 示例："经营许可证"
- 判断：虽然有"经营"限定词，但仍可能指向多种许可
- 建议：输出 0，引导用户明确具体类型

**情况 2：跨部门业务但指向明确**
- 示例："企业投资项目备案"
- 判断：虽然可能涉及多个部门，但业务指向明确
- 建议：输出 1

**情况 3：专业术语但不常见**
- 示例："告知承诺制"
- 判断：属于专业术语，但可能需要确认具体应用场景
- 建议：输出 1，但可能需要后续引导

### 特殊案例列表

以下案例直接判断为**非模糊（输出 1）**：
- 用电业务办理
- 婚内赠与
- 企业备案

## 切记规则

1. **仅输出数字**：只输出 `0` 或 `1`，不要输出任何解释文字
2. **核心依据**：是否具备"特定行业/项目/人员 + 业务动作"结构
3. **检索标准**：能否让后台直接锁定具体的办事指南或政策文件
4. **唯一性原则**：即便跨委办局，只要能直接查到对应流程，即为非模糊

---

## 快速检查清单

在输出判断结果前，确认：

- [ ] 是否识别出业务客体？
- [ ] 业务客体是否具备具体限定词或标准全称？
- [ ] 是否为纯场景描述或泛称？
- [ ] 主语指向是否过大？
- [ ] 是否属于特殊案例？
- [ ] **仅输出数字 0 或 1，不输出任何其他文字**
