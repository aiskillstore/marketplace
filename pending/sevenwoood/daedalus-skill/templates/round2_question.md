# 第2轮提问模板

**使用场景**：用户已回答第1轮问题，`turn_count == 2` 时使用。

**前置动作**：根据第1轮用户回答，先输出反馈块（亮点 + 留意点），然后再提出第2轮的问题。

**目标**：深挖用户对现有产品的喜好/厌恶，以及产品最让其兴奋的情感瞬间。

---

## 反馈块模板（必须）

在提问之前，先根据用户上一轮的回答输出以下结构：

```markdown
**👍 亮点**
- [从用户回答中提取1-2个有价值的洞察，给予具体肯定]

**⚠️ 留意点**
- [温和指出1个潜在风险或未澄清的矛盾，如果有的话；如果没有，可省略或写“暂无”]
```

### 亮点示例

- “你提到‘不想让用户有发帖负担’，这个痛点非常真实。”
- “你说喜欢Mars的暗调照片和寥寥几句文字，我能感受到你对审美有明确偏好。”
- “你描述的那个‘周末走到河边台阶坐下看夕阳’的场景，很有画面感。”

### 留意点示例

- “你既想要自动记录轨迹，又担心隐私，这两者需要平衡。”
- “你说讨厌网红店，但‘受欢迎’的定义可能会自然导向大众化，这点可以再想想。”
- “目前你还没有提到任何现有的竞品，这可能会让你重复发明轮子。”

------

## 第2轮提问内容

输出反馈块后，继续输出以下问题：

```markdown
为了更清楚你的想法，我再问两个小问题：

### 1. 你提到的 [从用户回答中提取的某个产品/工具] ，有没有什么让你觉得“要是它能怎样就好了”的地方？

（比如：功能再多一点？少一点？界面更简洁？更省事？）

### 2. 你心里理想的这个产品，最让你兴奋的一个瞬间是什么？

（比如：打开看到一张暗调照片？系统自动帮我串成一条路线？朋友说“这地方真不错”？）
```

## 变体处理

### 如果用户第1轮没有提到任何具体产品

- 将第一个问题改为：

  > “你之前有没有用过某个App或工具，让你觉得‘接近但不够好’？如果没有，也可以说说你讨厌的某个产品。”

### 如果用户第1轮回答非常简短（如只回答了1个问题）

- 反馈块后，可以追加一个提醒：

  > “为了帮你更好地聚焦，我们重点聊一下第2个问题：你心里特别喜欢的App或工具是哪个？喜欢它哪一点？”

- 然后继续上述第2轮问题。

------

## 示例对话

**用户（第1轮回答）**：

> 我经常周末想找个地方逛逛，但网上推荐全是网红店。我喜欢Mars，它的图片和文字很有感觉，可惜倒闭了。我希望打开产品就能看到一张暗调照片和一行字，我就知道这是我的菜。

**代达罗斯（第2轮输出）**：

```markdown
**👍 亮点**
- 你描述的场景非常具体：“周末想找个地方逛逛但被网红店轰炸”，这是很多人的真实痛点。
- 你从Mars提炼出的“暗调照片+一行字”的审美确认感，是一个很好的设计锚点。

**⚠️ 留意点**
- Mars倒闭可能与高客单价和小众用户量有关，你的产品需要考虑如何平衡调性和可负担性。

为了更清楚你的想法，我再问两个小问题：

### 1. 你提到的 Mars，有没有什么让你觉得“要是它能怎样就好了”的地方？
（比如：消费更低？用户更多？自动推荐路线？）

### 2. 你心里理想的这个产品，最让你兴奋的一个瞬间是什么？
（比如：打开后看到一张暗调照片？系统自动串成一条可跟随的路线？还是发现一个没有人知道的角落？）
```

## 注意事项

- 反馈块中的“留意点”不要过多（1-2条即可），以免打击用户积极性。
- 如果用户明显没有风险点（比如想法很朴素），可以写“⚠️ 留意点：暂无”。
- 第2轮问题不超过2个，保持轻量。
