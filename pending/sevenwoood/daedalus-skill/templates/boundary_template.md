# 边界定义模板（boundary_template.md）

**使用场景**：
- 第3轮输出后，引导用户明确功能边界（必须 / 延后 / 不做）。
- 后续对话中，当用户提出新功能且可能超出当前边界时，再次使用。

**目标**：帮助用户控制范围，避免V1版本功能膨胀。

---

## 模板版本1：第3轮边界确认（初次使用）

在 `round3_triggers.md` 的第五部分已包含，此处为独立可复用的版本：

```markdown
**🔲 边界定义**

为了让项目可控，请明确以下三类功能：

1. **当前版本（V1）必须有的功能**（最多3个）  
   请列出：  
   - 
   - 
   - 

2. **可以延后做的功能**（未来版本）  
   请列出（可选）：

3. **明确不做的范围**（帮助聚焦）  
   请列出（可选）：

（回复时可直接用数字列表或简短描述）
```

## 模板版本2：后续边界变更提醒

当用户在第4轮及之后提出新功能，且该功能不在当前边界内时，使用以下模板：

```markdown
**🔲 边界提醒**

你提到了一个新功能：[用户提出的功能]。

根据之前的边界定义，你当前V1版本已有以下必须功能：
- [must_have_1]
- [must_have_2]
- [must_have_3]（如果已有3个）

请选择如何处理：
- **A. 替换**：用新功能替换现有的某个V1功能（请说明替换哪一个）
- **B. 延后**：将新功能放入“延后列表”，未来版本再做
- **C. 不做**：明确不实现这个功能

如果当前V1功能不足3个，也可以直接选择 **D. 添加**：将新功能加入V1必须功能。
```

## 模板版本3：边界总结与确认

当用户完成边界定义后，输出确认摘要：

```markdown
**✅ 边界确认摘要**

- **V1必须功能（不超过3个）**：
  1. [功能1]
  2. [功能2]
  3. [功能3]

- **延后功能**：
  - [功能A]（如有）
  - [功能B]

- **不做范围**：
  - [范围X]（如有）
  - [范围Y]

> 请确认以上边界是否符合你的预期？回复“确认”或提出修改。
```



------

## 边界维护规则（供技能参考）

- **数量限制**：V1必须功能严格不超过3个。如用户试图添加第4个，强制要求替换或延后。
- **不做范围的价值**：即使为空也没关系，但建议至少列出1-2个“明确不做”的事项，帮助聚焦。
- **更新时机**：每轮对话后，如果用户主动修改边界，需重新输出确认摘要。
- **记录位置**：存储在状态变量 `must_haves`、`postponed`、`out_of_scope` 中。

## 示例对话

**用户（第3轮后）**：

> V1必须有：1. 查看附近小众店铺 2. 一键生成路线 3. 收藏店铺
> 延后：评论功能、社交分享
> 不做：付费功能、广告

**代达罗斯**：

> **✅ 边界确认摘要**
>
> - **V1必须功能**：
>   1. 查看附近小众店铺
>   2. 一键生成路线
>   3. 收藏店铺
> - **延后功能**：评论功能、社交分享
> - **不做范围**：付费功能、广告
>
> 请确认以上边界是否符合你的预期？回复“确认”或提出修改。
