# 分析框架与方法论

本文档定义 Prism-Skill 在分析过程中可调用的学科适配规则、视角选择逻辑以及常用分析模型。

---

## 一、动态学科适配规则

### 1. 学科分类库（不限于此，动态扩展）

| 类别       | 包含学科                                     | 典型触发词                   |
| ---------- | -------------------------------------------- | ---------------------------- |
| 自然科学   | 物理学、化学、生物学、医学、地理学、天文学   | 病毒、药物、气候、基因       |
| 社会科学   | 经济学、政治学、社会学、心理学、法学、教育学 | 利率、选举、阶层、焦虑、判决 |
| 形式科学   | 数学、统计学、逻辑学、计算机科学             | 概率、算法、证明、模型       |
| 人文学科   | 历史学、哲学、语言学、艺术学                 | 起源、意义、语法、风格       |
| 工程与技术 | 各类工程、建筑、材料、电子                   | 设计、结构、芯片、自动化     |

### 2. 选择优先级
- **用户明确指定**（如“从医学角度”）→ 优先使用该学科。
- **用户未指定** → 根据问题中的实体或动词，识别最相关的1-3个学科。
- **置信度低**（多个学科均相关）→ 在第一轮分析末尾追加提问，引导用户选择。

### 3. 数量控制
- 至少1个，最多3个。避免过度堆砌导致分析臃肿。

## 二、主体/视角适配规则

### 1. 层级分类

| 层级 | 示例主体                     | 典型问题方向                   |
| ---- | ---------------------------- | ------------------------------ |
| 微观 | 个人、家庭、企业、组织       | 个人感受、家庭预算、公司利润   |
| 中观 | 城市、行业、机构、社区       | 产业影响、区域发展、机构效率   |
| 宏观 | 国家、区域联盟、国际体系     | 政策制定、地缘关系、全球趋势   |
| 特殊 | 生物物种、生态系统、技术系统 | 物种演化、生态平衡、系统稳定性 |

### 2. 选择规则
- 优先使用问题中明确提及的主体（如“我的房贷” → 微观）。
- 否则，选择最能解释问题因果链的1-2个层级。
- 对于复杂问题，可跨层级组合（如微观个人行为 + 宏观政策影响）。

## 三、推荐分析模型（按需调用）

### 1. PESTLE 分析
- **用途**：全面扫描外部环境因素。
- **维度**：政治、经济、社会、技术、法律、环境。
- **输出形式**：表格或列表。

### 2. 六顶思考帽
- **用途**：多视角审视同一问题，避免偏见。
- **帽子**：白（事实）、红（直觉）、黑（风险）、黄（价值）、绿（创意）、蓝（过程）。
- **输出形式**：列表或段落。

### 3. 因果回路图（文本版）
- **用途**：展示变量之间的正向/负向反馈。
- **符号**：`A ↑ → B ↑`（正向），`A ↑ → B ↓`（负向）。
- **输出形式**：文本流程图（缩进或箭头链）。

### 4. 三角验证
- **用途**：对关键结论寻找至少2个不同学科或来源的支撑证据。
- **执行时机**：当用户触发“真实性优先”模式，或结论存在争议时。

### 5. SWOT 交叉分析
- **用途**：结合内部因素（优势、劣势）与外部因素（机会、威胁）。
- **输出形式**：2×2 矩阵表。

---

## 四、分析流程示例

1. **接收输入** → 识别关键词 → 动态选择学科与视角。
2. **判断宽泛程度** → 若模糊则追问（最多3轮）。
3. **调用适配模型**（如事件分析用 PESTLE，决策问题用 SWOT）。
4. **交叉验证**（必要时使用三角验证）。
5. **结构化输出**（柔和标题 + 表格/流程图 + 小结 + 后续引导）。

---

## 五、注意事项

- 模型选择应自然，不强行套用。如果简单问题用因果回路图显得冗余，则直接列表说明。
- 文化敏感性：分析不同国家政策时，避免单一西方框架，可同时使用该国官方表述与国际观察。
- 动态更新：本框架可随技能迭代扩展新的学科或模型。
