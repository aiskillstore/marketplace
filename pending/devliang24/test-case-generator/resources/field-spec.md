# 测试用例字段规范

本文档定义了测试用例的标准字段格式，共 15 个字段（列）必须按顺序输出。
注意：字段列必须完整，但部分字段允许为空：`实际结果` 必须留空，`需求ID` 在需求文档未提供时可留空；其他字段如无内容请填写“无”。

## 字段列表

| 序号 | 字段名称 | 字段说明 |
|------|----------|----------|
| 1 | 用例目录 | 测试用例所属的目录或平台名称 |
| 2 | 模块 | 测试用例所属的功能模块 |
| 3 | 功能 | 测试用例对应的具体功能点 |
| 4 | 用例名称 | 测试用例的标题或名称 |
| 5 | 前置条件 | 执行测试用例前需要满足的条件 |
| 6 | 用例步骤 | 测试用例的执行步骤，详细描述操作流程 |
| 7 | 测试数据 | 测试用例执行时使用的测试数据，如输入参数、测试文件、配置信息等 |
| 8 | 预期结果 | 执行测试用例后应该得到的结果 |
| 9 | 实际结果 | 执行测试用例后实际得到的结果（测试执行时填写） |
| 10 | 用例类型 | 测试用例的类型，仅限：功能测试 / 安全测试 / 兼容性测试 / 易用性测试 |
| 11 | 用例类型（正向/异常） | 测试用例的类型分类，用于区分正向测试用例和异常测试用例 |
| 12 | 用例状态 | 测试用例的当前状态，如：待测试、已通过、已失败等 |
| 13 | 用例等级 | 测试用例的优先级等级，如：高、中、低 |
| 14 | 需求ID | 关联的需求编号或ID |
| 15 | 创建人 | 创建该测试用例的人员姓名 |

## 字段详细说明

### 1. 用例目录
- **说明**：测试用例所属的目录或平台名称
- **示例**：AI模型平台/推理服务、电商系统/订单管理

### 2. 模块
- **说明**：测试用例所属的功能模块
- **示例**：模型推理、模型训练、模型网关、用户管理、商品管理

### 3. 功能
- **说明**：测试用例对应的具体功能点
- **示例**：推理任务提交、训练任务创建、用户登录、商品搜索

### 4. 用例名称
- **说明**：测试用例的标题或名称，应简洁明了
- **格式建议**：[场景]-[操作]-[预期]
- **示例**：正常场景-提交推理任务-成功返回任务ID

### 5. 前置条件
- **说明**：执行测试用例前需要满足的条件
- **示例**：
  - 用户已登录系统
  - 测试数据已准备完毕
  - 服务器运行正常
  - 已选择模型版本 v1.0
- **填写规则**：如无前置条件，请填写“无”。

### 6. 用例步骤
- **说明**：测试用例的执行步骤，详细描述操作流程
- **格式建议**：使用编号列表，每步一行
- **示例**：
  ```
  1. 打开推理任务页面
  2. 选择模型版本 v1.0
  3. 上传测试文件 test.csv
  4. 点击"提交任务"按钮
  5. 等待任务执行完成
  ```

### 7. 测试数据
- **说明**：测试用例执行时使用的测试数据
- **示例**：
  - 输入参数：{"model": "v1.0", "batch_size": 32}
  - 测试文件：test_data.csv (100条记录)
  - 配置信息：timeout=30s
- **填写规则**：如无测试数据，请填写“无”。

### 8. 预期结果
- **说明**：执行测试用例后应该得到的结果
- **格式建议**：明确、可验证
- **示例**：
  - 系统返回任务ID：task_123456
  - 页面显示"任务提交成功"提示
  - 任务状态为"运行中"

### 9. 实际结果
- **说明**：执行测试用例后实际得到的结果
- **填写时机**：测试执行时填写
- **生成时**：保持为空

### 10. 用例类型
- **说明**：测试用例的类型
- **可选值（仅限以下 4 类）**：
  - 功能测试：验证功能是否按预期工作
  - 安全测试：验证安全机制
  - 兼容性测试：验证跨平台、跨浏览器兼容性
  - 易用性测试：验证用户体验
- **注意**：不得输出上述 4 类以外的其他类型。

### 11. 用例类型（正向/异常）
- **说明**：区分正向测试用例和异常测试用例
- **可选值**：
  - **正向**：验证正常功能流程，输入合法数据，验证系统按预期工作
  - **异常**：验证错误处理、边界条件、异常场景等，输入非法数据或极端情况

### 12. 用例状态
- **说明**：测试用例的当前状态
- **默认值**：待测试
- **可选值**：
  - 待测试：尚未执行
  - 已通过：测试通过
  - 已失败：测试失败
  - 阻塞：被其他问题阻塞
  - 跳过：本次测试跳过

### 13. 用例等级
- **说明**：测试用例的优先级等级
- **可选值**：
  - **高**：核心功能、高频使用场景、影响重大
  - **中**：常规功能、中频使用场景
  - **低**：边缘功能、低频使用场景、影响较小

### 14. 需求ID
- **说明**：关联的需求编号或ID
- **示例**：REQ-2024-001、JIRA-1234
- **注意**：如果需求文档中未提供，可留空

### 15. 创建人
- **说明**：创建该测试用例的人员姓名
- **默认值**：AI助手
- **填写规则**：由本 Skill 生成的测试用例固定填写“AI助手”。

## 输出格式要求

生成的测试用例必须严格按照以上 15 个字段的顺序输出，不可遗漏任何字段。

### Excel 格式要求

- 第一行为表头（15个字段名称）
- 表头格式：粗体、背景色、居中对齐
- 数据行从第二行开始
- 每行一条测试用例
- 单元格格式：文字左对齐、自动换行
- 列宽自动调整以适应内容
