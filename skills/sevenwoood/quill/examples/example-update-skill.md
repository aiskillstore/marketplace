---

## 场景概述

**用户目标**：用户已拥有一个“余华早期风格”的 Skill。后来找到了余华后期的两篇作品，希望将 Skill 升级，加入后期风格特征。

**操作路径**：
1. 确认已拥有的 Skill
2. 投喂新文章（增量更新）
3. 系统检测并生成《风格漂移报告》
4. 用户选择更新范围
5. 系统执行增量更新
6. 验证更新效果

# 余华后期 写作引擎 V1.0

## 🚫 强制禁忌

- 无特定禁忌（用户未指定）

## 🔥 核心指令

以温和的叙事距离回望命运的流转，用稍长的句法和适度内视角，呈现记忆与告别的重量。

## 📝 强制性语言规则

1. **词汇**：
   - 高频词：记忆、告别、城市、医院、时间、归来
   - 专属意象：雨夜、空房间、旧照片、走廊、黄昏
   - 禁用万能词：很好、非常、特别、十分
2. **句式**：
   - 平均句长：18-24字（较早期明显拉长）
   - 句法特征：短句与长句交错，节奏更舒缓
   - 逗号密度：较高（每千字50-65个）
   - 分段习惯：段落稍长，平均3-5行
3. **修辞**：
   - 修辞密度：每百字 1-2 次
   - 主要类型：暗喻为主，偶有明喻
   - 修辞功能：用于营造氛围，而非揭示残酷

## 🎥 叙事视角

- 视角类型：第三人称有限（有节制地进入人物内心）
- 心理距离：中（较早期的“远”拉近）
- 表达倾向：Show 70% / Tell 30%
- 心理描写配额：每千字 1-2 句直接心理描写

## 🎵 节奏律动

- 紧张场景：句长缩短至10-15字
- 舒缓场景：句长延长至20-28字
- 情绪转换：通过句长渐变完成，较少使用单句独立成段的突变

- 未启用（本次蒸馏未分析此维度）

## 要点总结

| 环节     | 用户操作                  | 系统行为                          |
| :------- | :------------------------ | :-------------------------------- |
| 触发更新 | 输入“更新Skill：[新文章]” | 对比新旧文本，计算各维度差异      |
| 接收报告 | 阅读《风格漂移报告》      | 输出差异对比 + 建议更新范围       |
| 选择范围 | 回复数字或具体说明        | 根据选择执行增量更新或生成新Skill |
| 验证结果 | 切换至不同Skill并改写     | 对比新旧Skill的输出差异           |

## 关键特征

1. **自动检测漂移**：用户无需自行判断“新文章和旧Skill有没有差异”，系统自动完成。
2. **四种更新路径可选**：仅更新单维度 / 全部覆盖 / 保留旧版生成新版 / 仅更新意象词库。
3. **保留还是升级由用户决定**：系统不替用户做“哪个版本更好”的价值判断。
4. **切换验证**：用户可快速切换新旧Skill，对比同一段文字的改写效果，验证更新是否达到预期。
5. **版本号自动递增**：Skill卡标题中的 V1.0 在更新后自动更新为 V1.1、V2.0 等（若用户选择覆盖）。
