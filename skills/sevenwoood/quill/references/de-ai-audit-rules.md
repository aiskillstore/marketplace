# 去AI味审计规则库

本文档定义了 Quill（羽笔）在风格化改写过程中强制执行的 **去AI味审计规则**。所有规则在生成 Skill 卡时自动叠加，在改写执行时强制校验。

> ⚠️ **重要**：本文档为后台技术参考，仅供开发者/贡献者查阅。所有审计规则对用户 **完全黑箱化** ——AI 在后台执行审计，用户仅看到最终的《修改注记》中“去AI味”维度的简要说明。

---

## 规则总览

| 层级     | 规则数量               | 校验方式 | 违规处理 |
| -------- | ---------------------- | -------- | -------- |
| 词汇层   | 3类禁用词库 + 替换规则 | 自动扫描 | 强制替换 |
| 句法层   | 5类禁用结构 + 长度限制 | 自动扫描 | 强制改写 |
| 标点层   | 3类限制规则            | 自动计数 | 强制削减 |
| 修辞层   | 2类限制规则            | 自动扫描 | 强制调整 |
| 最终检测 | 活人感测试             | 人工校验 | 退回重写 |


## 一、词汇层审计规则

### 1.1 绝对禁用词库（AI高频塑料词）

以下词汇在所有改写输出中 **绝对禁止出现**：

| 类别         | 禁用词                                           | 说明                   |
| ------------ | ------------------------------------------------ | ---------------------- |
| 模板化连接词 | 综上所述、总而言之、不可否认、毋庸置疑、众所周知 | 教科书面具，机器人最爱 |
| 万能弱动词   | 进行（如“进行分析”）、开展、推进、深化、加强     | 用精准动词替代         |
| 空洞形容词   | 很好、非常、十分、特别、极其、相当、颇为         | 用具体描述替代         |
| 官僚腔名词   | 赋能、抓手、深耕、闭环、逻辑、场景、痛点、打法   | 互联网黑话，远离人话   |
| 书面化副词   | 值得注意的是、需要指出的是、必须承认的是         | 故作深沉，实则废话     |
| 伪连接词     | 换言之、也就是说、与此同时                       | 过度解释，低估读者     |
| 冗余修饰     | 真正意义上的、某种程度上的、在某种意义上         | 添加零信息量           |

### 1.2 强制替换规则

| 禁用写法                | 替换为           | 说明             |
| ----------------------- | ---------------- | ---------------- |
| 笔者                    | 我               | 拒绝第三人称伪装 |
| 然而                    | 但 / 可是        | 更口语，更直接   |
| 因此                    | 所以 / 于是      | 拉近逻辑距离     |
| 此外 / 另外             | 而且 / 还有      | 更像日常说话     |
| 进行 + 动词（进行分析） | 直接动词（分析） | 去掉冗余前缀     |
| 对XX进行XX              | 直接动词XX       | 简化句式         |
| 具有XX性                | 直接描述XX       | 拒绝抽象后缀     |
| 做出了XX                | 直接动词XX       | 去掉傀儡动词     |

### 1.3 限定词缩减规则

目标：将每百字的限定词（非常、很、比较、相对、几乎、似乎、可能、大概）数量控制在 **≤ 3 个**。

> **原因**：人类写作中限定词频率远低于AI。过多的“可能”、“似乎”会让文章显得犹豫、缺乏确定感。


## 二、句法层审计规则

### 2.1 禁用句式结构

| 禁用句式                     | 示例                           | 替换方案                |
| ---------------------------- | ------------------------------ | ----------------------- |
| 以“X而且Y”开头的并列结构     | “这个理念既深刻而且具有普适性” | “这个理念深刻，也普适”  |
| 以“试想一下”开头的提问式引导 | “试想一下，如果你是…”          | “如果你是…”（直接切入） |
| “你有没有想过”类疑问句开头   | “你有没有想过为什么…”          | “为什么…”（直接提问）   |
| 以“如果我们”开头的条件假设   | “如果我们仔细分析就会发现…”    | “细看会发现…”           |
| 并列的“不仅…而且…”结构       | “不仅是一本书，而且是一个时代” | “是书，也是时代”        |
| 复杂嵌套从句（3层以上）      | “那个我曾经在年轻时…”          | 拆分或重排              |

### 2.2 句长限制规则

| 场景                 | 规则                            | 说明                                               |
| -------------------- | ------------------------------- | -------------------------------------------------- |
| 通用改写             | 平均句长 ≤ 25 字                | 若原始风格强制长句（如普鲁斯特），允许例外但需标注 |
| 短句风格（如海明威） | 平均句长 ≤ 15 字                | 极简风格强制要求                                   |
| 单句上限             | 任何单句不得超过 35 字          | 超出则强制拆分                                     |
| 句长多样化           | 每100字内至少包含1句≤10字的短句 | 防止全是中长句的“均匀感”                           |

### 2.3 段落首句结构限制

| 规则                         | 说明                             |
| ---------------------------- | -------------------------------- |
| 禁止每段首句为“中心句”       | 这是教科书/八股文的典型AI特征    |
| 段落首句允许以“具体动作”开头 | “他推开门…” vs “这个现象说明了…” |
| 段落首句允许以“具体物象”开头 | “窗外的雨…” vs “综上所述…”       |


## 三、标点层审计规则

### 3.1 破折号（—）使用限制

| 规则           | 阈值                            |
| -------------- | ------------------------------- |
| 破折号使用频率 | 每千字 ≤ 2 次                   |
| 违规判定       | > 2 次/千字 → 强制削减至 ≤ 2 次 |

> **原因**：破折号是AI写作的“代名词”之一。人类写作中破折号使用频率远低于AI。

### 3.2 省略号（……）使用限制

| 规则           | 阈值                            |
| -------------- | ------------------------------- |
| 省略号使用频率 | 每千字 ≤ 1 次                   |
| 违规判定       | > 1 次/千字 → 强制削减至 ≤ 1 次 |

### 3.3 感叹号使用限制

| 规则           | 阈值                                       |
| -------------- | ------------------------------------------ |
| 感叹号使用频率 | 每千字 ≤ 3 次                              |
| 违规判定       | > 3 次/千字 → 强制削减至 ≤ 3 次            |
| 特殊场景       | 对话中可适度放宽，但需在《修改注记》中标注 |


## 四、修辞层审计规则

### 4.1 比喻使用限制

| 规则         | 阈值                                                         |
| ------------ | ------------------------------------------------------------ |
| 比喻密度上限 | 每百字 ≤ 3 次                                                |
| 违规判定     | > 3 次/百字 → 强制削减至 ≤ 3 次                              |
| 特殊场景     | 若蒸馏源风格本身以“通感/密集比喻”为特征（如张爱玲），上限可放宽至每百字 ≤ 5 次，但需在Skill卡中标注例外 |

### 4.2 排比与重复结构限制

| 规则         | 阈值                                    |
| ------------ | --------------------------------------- |
| 排比结构频率 | 每500字 ≤ 1 组                          |
| 违规判定     | > 1 组/500字 → 强制拆分或重组           |
| 特殊场景     | 散文诗/演讲风格可适度放宽，但需标注例外 |


## 五、最终检测：活人感测试

### 5.1 测试流程

改写完成后，AI必须对输出文本执行以下 **三道自检**：

#### 测试一：大声朗读测试
> **读一遍整段文字，问自己：这段话是否可能出自一个真实人类的即兴口述？**

- 若答案为 **是** → 通过
- 若答案为 **否** → 退回重写（调整句式结构和用词）

#### 测试二：维基百科测试
> **扫描整段文字，是否存在“维基百科式”的客观总结句？**

- 若存在（如“这种现象反映了…”）→ 删除或替换为具体描述
- 若不存在 → 通过

#### 测试三：代词密度测试
> **检查每段中“我们”、“人们”、“大家”等泛指代词的出现频率。**

- 若每段 ≥ 2 个泛指代词 → 判定为过度抽象，替换为具体主语
- 若每段 < 2 个 → 通过

### 5.2 终审清单

执行改写后，隐式完成以下检查：

| 检查项       | 通过条件                                 |
| ------------ | ---------------------------------------- |
| 禁用词扫描   | 未发现任何“绝对禁用词库”中的词汇         |
| 句长合规     | 平均句长 ≤ 25字（或符合Skill卡目标范围） |
| 破折号审计   | 每千字 ≤ 2 次                            |
| 段落首句审计 | 不存在中心句开场                         |
| 泛指代词审计 | 每段 < 2 个泛指代词                      |
| 大声朗读测试 | 听起来像人在说话                         |


## 六、AI味污染源检测

### 6.1 定义

在蒸馏执行之前，Quill 会对投喂的源文本执行一次 **“AI味污染源检测”**，判断用户提供的素材本身是否由AI生成。若源文本本身就带有AI味，蒸馏时会污染风格模型。

### 6.2 检测特征（后台标记）

以下特征任意命中 3 条以上，即判定源文本存在AI味：

| 检测特征                     | 说明                         |
| ---------------------------- | ---------------------------- |
| 存在“综上所述”类模板化连接词 | ≥ 2 次/千字                  |
| 平均句长 > 30 字             | 且句长标准差 < 15%（均匀感） |
| 每段首句为中心句             | ≥ 80% 段落符合               |
| 缺乏具体人称主语             | “我们”、“人们”过度使用       |
| 段落结构固定                 | 每段 3-5 句，格式统一        |
| 比喻密度极低或极高           | < 0.5 次/百字 或 > 6 次/百字 |

### 6.3 处理方式

若判定源文本存在AI味：

- 在蒸馏执行时，**排除**被标记的“AI味特征”（模板化连接词、均匀句长、中心句结构），仅提取**真人表达部分**。
- 在生成的 Skill 卡中，【强制禁忌】模块追加一条：

> ⚠️ 源文本存在AI写作痕迹。蒸馏时已排除模板化特征（连接词、均匀句长、中心句结构），仅提取真人表达部分。


## 七、规则应用方式

### 在蒸馏阶段

- 去AI味规则 **自动叠加** 至所有生成的Skill卡中，无需用户操作。
- 规则以独立模块 **【🔴 去AI味审计规则】** 形式出现在Skill卡中。

### 在改写阶段

- AI严格执行所有审计规则。
- 改写完成后，若任一规则被违反，AI需**自动重写**通过审计后才能输出。
- 最终输出中，【修改注记】模块需包含“去AI味”维度的简要说明：

- **去AI味**：替换“综上所述”→“总之”，削减破折号3处→1处，句长从32字压缩至22字

```text
### 豁免机制

- 若用户输入中包含“保留AI味”或“不需要去AI化”，则跳过本审计规则库。
- 若蒸馏源风格本身以“书面化/学术化”为特征（如学术论文），部分规则（如平均句长 ≤25字）自动放宽，以源风格为准。
```
