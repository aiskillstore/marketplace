# 🏛️ Daedalus Skill · 代达罗斯

**对话式产品孵化技能** —— 从模糊想法到可执行产品方案的完整引导引擎。通过结构化提问、风险提示、竞品对照和边界定义，帮助任何人（无论是否懂技术）将产品念头转化为清晰的产品需求文档（PRD）。

![Daedalus Banner](assets/daedalus-skill_banner.svg)

> "你不需要会造迷宫，你只需要告诉我要去哪里。"

## 📖 简介

代达罗斯（Daedalus）是一个遵循 [Agent Skills 协议](https://skills.sh) 的对话式产品孵化技能。它能够：

- 从**零基础用户的模糊想法**出发，通过结构化提问逐轮具象化产品概念；
- **每轮提供反馈**（亮点 + 留意点），建立信任的同时引导用户自我修正；
- 第3轮自动输出**项目门槛评估、竞品参考、合规风险提示**和**立项判断**；
- 引导用户明确 **V1 功能边界**（必须 ≤ 3 个），防止范围蔓延；
- 最终生成完整的**产品需求文档（PRD）**，并附带**分专业/零代码的工具链推荐**；
- 支持**进度保存与恢复**、**降级策略**（用户回复过短时自动切换菜单模式）。

## ✨ 主要特性

| 特性                  | 说明                                                         |
| --------------------- | ------------------------------------------------------------ |
| 🗣️ 承接模糊想法        | 不评判、不否定，用感性开放的问题引导用户具象化               |
| 🔄 结构化轮次对话      | 12 轮对话流程，每轮不超过 3 个问题，逐步深入                 |
| 💰 项目门槛评估        | 第 3 轮自动评估技术/资金/时间成本，提供轻量/中等/完整三条路径 |
| 📚 竞品参考推荐        | 基于关键词匹配内置竞品库，提供差异化分析                     |
| ⚖️ 合规 + 安全风险提示 | 从法律/伦理和技术/产品设计两个维度主动提醒风险               |
| 📋 边界定义            | 明确 V1 必须功能（≤ 3 个）、延后功能、不做范围               |
| 📄 结构化 PRD 输出     | 9 个核心章节的 Markdown 文档，可直接交给 AI 编程工具         |
| 🛠️ 工具链推荐          | 分专业工具和零代码工具两张表，适配不同技术背景               |
| 💾 进度保存与恢复      | 支持"保存退出"并后续继续，最大 12 轮自动结束                 |
| 🧩 降级策略            | 用户回复过短时自动切换为二选一菜单模式                       |

## 📦 安装

本技能符合 **Skills CLI** 规范，可通过以下任意方式安装：

### 方式一：通用 Skills CLI（推荐）

```bash
npx skills add sevenwoood/daedalus-skill --skill daedalus-skill
```



### 方式二：GitHub Skills CLI

```bash
gh skill install sevenwoood/daedalus-skill daedalus-skill
```



### 方式三：手动安装

```bash
git clone https://github.com/sevenwoood/daedalus-skill.git
# 将 daedalus-skill 文件夹复制到以下目录之一：
# - ~/.cursor/skills/
# - ~/.claude/skills/
# - ~/.codex/skills/
# 或者直接作为独立项目使用
```



> 若平台不支持完整文件结构，可将 `instructions.md` 作为 System Prompt 植入，其余文件作为参考。

## 🚀 快速开始

**输入示例**：

"我想做一个分享小众好店的 App，周末经常不知道去哪逛，但是小红书上的网红店又太假了。"

**技能输出示例**（第 3 轮门槛评估）：

```text
💰 项目门槛评估
根据你的描述，这个产品可能涉及：
- 后端服务器/数据库（如需要用户账号、存储轨迹）
- 地图 API 调用（超过免费额度可能收费）
- 云存储（如图片上传）

建议你选择一条路径：
1. 轻量级验证：成本 < 500 元，几天时间，无后台（如纯前端小程序、本地存储）
2. 中等完整度：成本几千元，几周时间，简单后端（如 Supabase、LeanCloud）
3. 全功能产品：成本万元以上，月固定费用，需开发团队
```



**最终 PRD 输出**：当用户选择"开始做"或对话结束时，代达罗斯会生成包含以下 9 个章节的完整产品需求文档：

1. 核心痛点与场景
2. 用户画像
3. 设计原则（喜欢/讨厌）
4. 功能边界
5. 非功能需求（门槛评估结果）
6. 竞品对照
7. 风险提示摘要
8. 推荐的 V0.1 行动方案
9. 工具链推荐

## ⚠️ 使用注意事项

- **最佳使用模型**：建议使用 GPT-4、Claude-3、Gemini 1.5 Pro 或更高版本以获得完整体验。
- **能力边界**：代达罗斯不编写实际代码、不做 UI 视觉设计、不提供法律正式意见、不保证产品成功或商业变现。
- **风险提示**：所有合规与安全风险提示仅供参考，不同地区法规可能不同，建议针对具体项目咨询专业人士。
