# 风险检查清单（risk_checklist.md）

本文档定义代达罗斯用于匹配用户功能的风险关键词库及对应的提示模板。

---

## 使用说明

- **扫描方式**：每轮对话后，扫描用户输入中的所有功能描述，与关键词库进行匹配。
- **去重机制**：已提醒过的风险类别记录在 `risk_reminded` 列表中，避免重复输出。
- **输出位置**：
  - 第3轮：输出所有匹配的合规风险（一次性卡片）。
  - 后续每5轮：只输出新增匹配的风险（未提醒过的）。
  - 用户主动要求“分析风险”：输出所有已匹配但尚未提醒的风险。

---

## 第一部分：合规风险（法律/伦理）

### 关键词库与风险说明

| 关键词              | 触发词示例                                 | 风险等级 | 提示模板（摘要）                                             |
| ------------------- | ------------------------------------------ | -------- | ------------------------------------------------------------ |
| 位置追踪            | 定位、轨迹、打卡、签到、附近的人、实时位置 | 高       | 需遵守《个人信息保护法》，必须明示告知、征得同意、提供关闭和数据删除功能。 |
| 用户上传内容（UGC） | 图片、评论、帖子、分享、发布、上传         | 中       | 需建立内容审核机制（关键词过滤+举报+第三方API），否则可能因违规内容导致下架。 |
| 匿名社交            | 匿名、悄悄话、私密、无登录                 | 高       | 易被滥用进行骚扰或违法信息传播，建议增加身份验证（如手机号绑定）或举报功能。 |
| 健康数据            | 心率、步数、减肥、体重、医疗、病症         | 高       | 涉及敏感个人信息，需更高级别的加密和合规声明，部分功能可能需要医疗器械认证。 |
| 金融相关            | 支付、红包、理财、借贷、转账、钱包         | 极高     | 需支付牌照或合规资质，个人开发者无法上线。建议使用微信支付/支付宝的合规接口。 |
| 未成年人            | 儿童、学生、青少年、18岁以下               | 高       | 需符合《未成年人保护法》，实施年龄验证、家长同意机制，不得收集不必要信息。 |
| 境外地图            | Google Maps、MapKit（境外）、境外地图      | 中       | 国内应用必须使用合规地图服务（高德、百度、腾讯），否则违法。 |
| 社交关系            | 好友、关注、粉丝、私信                     | 中       | 需提供隐私设置、拉黑举报功能，遵守《网络安全法》。           |

```markdown
- **[风险类别]**：[具体风险说明]

- **[风险类别]**：[具体风险说明]

> 免责声明：我们已经尽力帮你识别常见的法律、伦理风险，但不同地区的规定可能更新较快，建议你针对自己的项目具体咨询专业人士或自行查询最新法规。
```

## 第二部分：安全风险（技术/产品设计）

### 关键词库与风险说明

| 功能点       | 关键词                 | 风险描述                                       | 提示模板                                                     |
| :----------- | :--------------------- | :--------------------------------------------- | :----------------------------------------------------------- |
| 图片上传     | 上传图片、拍照、相册   | 云存储被滥用作图床，产生高额费用或传播非法内容 | 建议限制文件类型（jpg/png）、大小（≤5MB）、添加防盗链和图片审核API。 |
| 短信验证码   | 短信登录、验证码、注册 | 恶意刷量导致费用飙升，用户被骚扰               | 需设置频率限制（同一手机号1次/分钟）、图形验证码、IP黑名单。 |
| 用户数据存储 | 个人信息、密码、轨迹   | 数据泄露、未加密存储                           | 敏感信息必须加密存储（如bcrypt），传输使用HTTPS，定期备份。  |
| 位置记录     | 定位、轨迹、打卡       | 轨迹数据滥用                                   | 明确告知用途，提供删除功能，定期清理历史数据（如保留90天）。 |
| 第三方API    | API密钥、地图服务      | API密钥泄露导致盗刷                            | 密钥不能硬编码在前端，需在后端代理请求，设置调用频率限制。   |
| 用户身份伪造 | 冒充他人、盗用账号     | 冒充他人发布内容或查看隐私                     | 需要基本的身份校验（手机号绑定、邮箱验证），敏感操作要求二次验证。 |
| 未授权接口   | 数据遍历、越权         | 接口无鉴权，可遍历其他用户数据                 | 所有写操作和敏感读操作需验证用户身份（token），使用随机ID而非自增ID。 |

### 安全风险输出模板

当用户提出上述功能时，在对应轮次的反馈块中附加（放在“留意点”之后）：

```markdown
**🔒 安全风险提醒（产品经理角度）**  
你提到的 [功能] 需要注意：  
- [具体风险1]：[建议]  
- [具体风险2]：[建议]
```

## 第三部分：快速匹配逻辑（伪代码）

```python
def scan_risks(user_input, reminded_list):
    new_risks = []
    
    # 合规风险扫描
    for keyword, risk_info in compliance_keywords.items():
        if keyword in user_input and risk_info.category not in reminded_list:
            new_risks.append(risk_info)
    
    # 安全风险扫描（仅当用户输入包含明确功能描述时）
    for keyword, risk_info in security_keywords.items():
        if keyword in user_input and risk_info.category not in reminded_list:
            new_risks.append(risk_info)
    
    return new_risks
```

## 第四部分：风险提示示例（合并输出）

**合规风险示例**（第3轮）：

```markdown
**⚖️ 法律与伦理风险提示**

根据你描述的“用户可以在App内打卡签到并上传店铺照片”，系统识别到以下风险领域：

- **位置追踪**：记录用户签到地点需遵守《个人信息保护法》，必须弹窗告知、征得同意，并提供“关闭轨迹记录”功能。

- **用户上传内容**：上传的图片需进行内容审核（涉黄、涉政、广告），否则可能被应用商店下架。建议接入第三方审核API（如阿里云、腾讯云）。

> 免责声明：我们已经尽力帮你识别常见的法律、伦理风险，但建议你咨询专业人士或自行查询最新法规。
```



**安全风险示例**（用户提出“用户可上传图片”后的反馈块中）：

```markdown
**🔒 安全风险提醒（产品经理角度）**  
你提到的“用户上传店铺照片”功能需要注意：  
- **图床滥用**：攻击者可能上传非法图片或利用你的云存储作为图床，建议限制文件类型（jpg/png）、大小（≤5MB），并添加防盗链和图片审核。  
- **存储费用**：如果不加限制，恶意上传会产生高额云存储费用，建议设置每日上传次数限制（如每人每天最多5张）。
```
