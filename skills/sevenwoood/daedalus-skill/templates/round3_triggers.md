# 第3轮输出模板（三大模块 + 立项判断 + 边界确认）

**使用场景**：用户已回答第2轮问题，`turn_count == 3` 时使用。

**目标**：集中输出项目门槛评估、竞品参考、合规风险提示，然后给出立项判断并引导边界确认。

**输出顺序**（固定）：
1. 项目门槛评估（含路径选择提问）
2. 竞品参考（含差异化提问）
3. 合规风险提示（含免责声明）
4. 立项判断（清晰/模糊）
5. 边界确认（引导用户列出V1必须功能等）

**注意**：本模板内容较长，请按顺序逐块输出，每块之间用 `---` 分隔。

---

## 输出模板

### 1. 项目门槛评估

```markdown
**💰 项目门槛评估**

根据你之前的描述，这个产品可能涉及以下技术组件和成本：

- **后端/数据库**：如需用户账号、存储轨迹数据，需要服务器和数据库。
- **地图API**：调用地图服务可能产生费用（高德/百度地图免费额度通常足够初期）。
- **云存储**：如果允许用户上传图片，会产生存储和流量费用。
- **运维学习**：即使使用低代码方案，也需要花时间学习部署、备份等。

**建议你选择一条路径**（回复数字 1、2 或 3）：

1. **轻量级验证**：成本 < 500元，几天时间，无后台或纯前端（如本地存储、静态地图）。
2. **中等完整度**：成本几千元，几周时间，简单后端（如微信小程序云开发、Supabase免费额度）。
3. **全功能产品**：成本万元以上，月固定费用，需开发团队或深入学习后端。

请回复 1、2 或 3。
```

### 3. 合规风险提示

```markdown
**⚖️ 法律与伦理风险提示**

根据你描述的功能，系统匹配到以下需要关注的风险领域：

- **[位置追踪]**：如果产品涉及记录用户轨迹或打卡地点，必须遵守《个人信息保护法》，在首次使用时明确告知用户、征得同意，并提供关闭选项和数据删除功能。
- **[用户上传内容]**：如果允许用户上传图片、文字，需建立内容审核机制（关键词过滤+人工/第三方API审核+举报通道），否则可能因违规内容导致下架。
- **[地图合规]**：面向国内用户必须使用合规地图服务（高德、百度、腾讯），不得使用境外地图（如Google Maps）。

> 免责声明：我们已经尽力帮你识别常见的法律、伦理风险，但不同地区的规定可能更新较快，建议你针对自己的项目具体咨询专业人士或自行查询最新法规。

如需更详细的安全技术风险提示（如云存储滥用、验证码刷量等），请在后续对话中提出具体功能，我会专门提醒。
```
