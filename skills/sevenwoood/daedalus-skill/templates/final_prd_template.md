# 最终PRD输出模板（final_prd_template.md）

**使用场景**：用户选择“开始做”或主动结束对话时输出。

**目标**：生成结构化的产品需求文档，可直接保存为 `.md` 文件，或交给 AI 编程工具（如 v0.dev、Cursor）作为输入。

**输出格式**：严格遵循以下 Markdown 结构。

---

```markdown

> 用户项目代号：[可选]

## 9. 工具链推荐

### 专业工具（适合有开发经验的团队）

| 阶段 | 工具 | 说明 |
|------|------|------|
| 原型设计 | Figma | 专业设计工具，需学习 |
| 前端代码 | Cursor / Copilot | AI辅助编程，需懂基础代码 |
| 后端 | Supabase | 开源后端，需懂数据库 |

### 零代码 / AI友好工具（适合非技术用户）

| 阶段 | 工具 | 说明 |
|------|------|------|
| 原型/视觉 | Canva / 即时设计 | 拖拽式，无需代码 |
| 小程序/App | 微信小程序云开发 | 低代码，免费额度够初期使用 |
| AI写代码 | v0.dev / bolt.new | 直接描述需求生成前端代码，不需要懂编程 |
| 地图 | 高德地图API | 有详细文档，可复制示例代码 |

> 💡 如果你完全没有技术背景，建议从“零代码”工具开始，先做一个可点击的原型验证想法。

## 10. 下一步指导

- 打开 [v0.dev](https://v0.dev) 或 [bolt.new](https://bolt.new)，粘贴本PRD中的“功能边界”和“设计原则”部分，让AI生成前端代码。
- 如果需要后端，注册 [Supabase](https://supabase.com) 免费账号，按照教程创建数据库。

**祝你的产品从想法走向现实！**
```

## 使用说明

1. **填充内容**：代达罗斯需根据对话中收集的状态变量（`user_idea`、`likes_dislikes`、`must_haves`、`threshold_choice`、`risk_reminded` 等）自动填充各个章节。
2. **缺失处理**：如果某项信息用户未提供（如用户画像、竞品对照），可以留空或标注“（未提供）”。
3. **路径适配**：根据用户选择的门槛路径，调整“预估成本与时间”以及“推荐的V0.1行动方案”的具体内容。
4. **结束语附加**：在PRD文档之后，附加 `README.md` 中定义的结束语（鼓励语 + 技能意义）。
