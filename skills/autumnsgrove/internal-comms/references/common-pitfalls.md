# Common Pitfalls and How to Avoid Them

This guide covers common mistakes in internal communications and how to prevent them.

## 1. Information Overload

**Problem**: Too much communication, drowning out important messages

**Symptoms**:
- Declining open rates
- Complaints about "too many emails"
- Important messages getting missed
- People tuning out announcements

**Solutions**:
- Consolidate into digests
- Use tiered importance (need-to-know vs. FYI)
- Establish communication hierarchies
- Respect "no meeting" times
- Create clear signal-to-noise ratio
- Sunset unused channels

**Prevention**:
- Ask "Does this need to be a separate message?"
- Use appropriate channels (wiki vs. email vs. Slack)
- Set communication calendars
- Batch related updates

## 2. Burying the Lede

**Problem**: Most important information hidden in middle or end of message

**Symptoms**:
- Questions about info you already shared
- Low engagement with critical information
- Confusion about what action is needed
- Having to re-send with clearer subject

**Solutions**:
- Use inverted pyramid structure
- Write TL;DR at the top
- Use clear, specific subject lines
- Put action items in visible locations
- Use formatting to highlight key info

**Prevention**:
- Write summary first
- Ask "What does the reader most need to know?"
- Test your message on someone unfamiliar with topic
- Use the "so what?" test

## 3. Inconsistent Formatting

**Problem**: Each communication looks and reads differently

**Symptoms**:
- Readers don't know where to find information
- Takes longer to scan and understand
- Looks unprofessional
- Harder to create and write

**Solutions**:
- Create and use templates
- Establish style guide
- Use consistent tools and platforms
- Train team on formats

**Prevention**:
- Document your templates
- Review for consistency before sending
- Use automation where possible
- Make templates easy to access

## 4. Lack of Clear Action Items

**Problem**: Readers don't know what they need to do

**Symptoms**:
- Questions like "What should I do?"
- Missed deadlines
- No response to requests
- Confusion about ownership

**Solutions**:
- Always include explicit action items section
- Use format: Action | Owner | Due Date
- Tag people directly (@mentions)
- Follow up on action items

**Prevention**:
- Ask "What action am I requesting?"
- Make deadlines specific
- Assign single owners (not teams)
- Make action items visible (formatting)

## 5. Missing Context

**Problem**: Assumes too much background knowledge

**Symptoms**:
- Many clarifying questions
- Misunderstanding of purpose
- Confusion about relevance
- "Why are we doing this?"

**Solutions**:
- Include background section
- Link to previous communications
- Explain the "why"
- Define acronyms and jargon
- Provide relevant history

**Prevention**:
- Write for the least informed audience member
- Have someone unfamiliar review
- Include "Background" or "Context" section
- Link to related documents

## 6. Overly Technical Jargon

**Problem**: Using language audience doesn't understand

**Symptoms**:
- Blank stares in meetings
- Questions about terminology
- Miscommunication across teams
- Exclusion of non-technical stakeholders

**Solutions**:
- Define technical terms
- Use analogies and examples
- Create glossaries
- Provide multiple explanation levels
- Link to documentation

**Prevention**:
- Know your audience
- Write at appropriate level
- Use "explain like I'm 5" test
- Get feedback from diverse reviewers

## 7. Irregular Communication Cadence

**Problem**: No predictable schedule for updates

**Symptoms**:
- Surprise announcements
- Missed information
- Stakeholder confusion
- Lack of trust

**Solutions**:
- Establish regular schedules
- Send even when "nothing to report"
- Use calendar invites for recurring comms
- Set expectations for frequency
- Stick to the schedule

**Prevention**:
- Create communication calendar
- Set reminders for regular updates
- Batch create multiple updates
- Delegate when you're unavailable

## 8. One-Way Communication

**Problem**: No opportunity for feedback or questions

**Symptoms**:
- Confusion and rumors
- Resistance to changes
- Lack of engagement
- Feeling of disconnect

**Solutions**:
- Always include how to ask questions
- Hold follow-up Q&A sessions
- Monitor Slack for questions
- Conduct pulse surveys
- Create feedback mechanisms

**Prevention**:
- Build in two-way channels
- Respond to questions promptly
- Acknowledge feedback received
- Close the loop on actions taken

## 9. Ignoring Different Communication Styles

**Problem**: Using only one medium or format

**Symptoms**:
- Some people always seem uninformed
- Complaints about communication methods
- Unequal access to information
- Declining engagement

**Solutions**:
- Use multiple channels for important messages
- Offer sync and async options
- Provide written and verbal versions
- Accommodate different time zones
- Make content accessible (captions, transcripts)

**Prevention**:
- Consider audience diversity
- Offer multiple formats
- Ask about preferences
- Track which methods work best

## 10. Lack of Follow-Through

**Problem**: Communications without accountability or updates

**Symptoms**:
- Action items not completed
- Lack of closure on topics
- Decreased trust in communications
- "What happened with that?"

**Solutions**:
- Track action items explicitly
- Send follow-up updates
- Close the loop on initiatives
- Report on outcomes

**Prevention**:
- Set reminders for follow-ups
- Include follow-up dates in original message
- Use project tracking tools
- Establish accountability
